# 法務・ブランドガイドライン整理メモ

## 1. 対象と前提
- 本アプリは日本国内の個人向け限定公開（最大6名想定）だが、広告表示やSNSデータ連携を行うため、一般公開と同等の法令・プラットフォームポリシーを意識する。
- 広告はアプリ内にネイティブ/バナー形式で表示する想定。アフィリエイトリンクや飲食店のPR掲載を含む。
- 取得データ: SNS上の公開情報（Instagram等）、Google Places APIの口コミ、ユーザーのお気に入り情報や端末ID等。

## 2. 遵守が必要な主な日本法令
| カテゴリ | 関連法令 / ガイドライン | 対応ポイント |
| --- | --- | --- |
| 個人情報 | 個人情報保護法 (APPI) | 個人情報の定義に該当するデータ（メール、端末ID）を明示し、利用目的・第三者提供・保管期間をプライバシーポリシーに記載。 |
| 通信・ログ | 電気通信事業法、総務省ガイドライン | 位置情報やアクセスログの取得について利用目的を明示し、ユーザー同意を得る。 |
| 広告表示 | 景品表示法、薬機法、健康増進法 等 | 広告表記を「広告」「PR」などで明確化。飲食店情報は事実に基づき誤認を避ける。 |
| 著作権 | 著作権法 | SNSから引用する画像・テキストはライセンス条件に従い、引用元を表示。 |
| 特定商取引法 | 有料サービス・課金が発生する場合 | 課金予定があれば事業者情報、連絡先、特商法表示を整備。 |
| 電子契約 | 電子署名法・電子帳簿保存法 | ToS/プライバシーに同意した日時をログで保持。 |

## 3. 利用規約 (Terms of Service) 骨子
1. **サービス概要**: SNS/Googleの公開情報と広告を組み合わせた飲食店検索サービスであること、限定ユーザー向けであることを記載。
2. **利用条件**: 招待制である点、不正アクセスやスクレイピングを禁止、未成年の利用可否。
3. **ユーザー責任**: 自己責任での店舗利用、広告内容の最終確認義務、違反時のアカウント停止。
4. **禁止行為**: リバースエンジニアリング、無断転載、広告の不適切利用、他者情報の投稿など。
5. **知的財産**: アプリ内のUI/コードは運営者に帰属し、Google Maps等の外部データは各社の権利に従う旨。
6. **広告表記**: 広告/アフィリエイトリンクを含む場合には「広告」「PR」と明記し、クリックによる報酬が発生することを説明。
7. **保証の否認・免責**: 表示情報の正確性や可用性を保証しない。API変更や規制によりサービス停止する可能性。
8. **利用停止・終了**: 規約違反、APIポリシー違反、広告主の要請等でサービスを停止できること。
9. **準拠法・裁判管轄**: 日本法準拠、東京地方裁判所等を専属管轄に指定。

## 4. プライバシーポリシー骨子
| 項目 | 説明 |
| --- | --- |
| 収集する情報 | (1) ユーザー提供情報: ニックネーム、メール、招待コード (2) 自動取得: 位置情報、端末情報、アクセスログ、閲覧履歴 (3) 外部API由来: Instagram投稿ID、Google Reviews等（個人情報に該当しないように加工）。 |
| 利用目的 | 店舗推薦、広告配信最適化、利用状況分析、問い合わせ対応。広告に関してはターゲティング条件の開示。 |
| 第三者提供 | 広告ネットワークや分析ツールへの送信可否を明示。個人情報を外部へ提供する場合は同意または法令に基づくこと。 |
| 共同利用 | チーム/運用メンバーが閲覧する範囲を限定し、アクセス権限管理を説明。 |
| 保管期間 | アカウント削除から○日以内に関連データを消去する旨を記載。 |
| 開示・訂正・削除請求 | 問い合わせ窓口を設置し、手続を説明。 |
| Cookie / 同様技術 | 広告表示や利用分析のため Cookie/LocalStorage を利用する場合の記述。 |
| 安全管理措置 | 暗号化、アクセス制御、ログ監査などの概要。 |
| 改定 | 規約改定時の通知方法（アプリ内通知、メール等）と効力発生日。 |

## 5. 広告表示ポリシー
- バナーやリスト内広告には「広告」「PR」バッジを表示し、非広告コンテンツと区別。
- 飲食店・飲料の広告で医療/健康効果を謳う場合は薬機法・健康増進法の表現規制をチェックし、「個人の感想です」表記など適切な注意書きを付す。
- インセンティブ広告（来店で報酬）を導入する場合は景品表示法の景品額制限（一般懸賞: 商品価格の2%等）を確認。
- 広告配信者（運営者 or 第三者広告ネットワーク）を明示し、問い合わせ先を提示。
- 行動ターゲティングを行う場合は、行動履歴の収集・拒否方法（オプトアウト）を記載。

## 6. ブランドガイドライン（主要プラットフォーム抜粋）
| 企業/サービス | 主な遵守事項 |
| --- | --- |
| Google Maps Platform | ロゴ・帰属表示をUIに表示 (`© Google`)、地図タイルやPOIデータのキャッシュ禁止。利用規約: <https://developers.google.com/maps/terms> を参照。 |
| Instagram / Meta | Graph API利用時は Meta Platform Policy に従い、利用目的とデータ保持ポリシーを App Review で説明。スクリーンショット公開時は @username と出典を明記。 |
| Google Places Reviews | Places APIのレビューは最大5件まで取得。引用時はレビュアー名・投稿日を付し、長期保存はキャッシュ扱い。 |
| 広告ネットワーク (例: Google AdSense) | アプリ内広告の配置ポリシーを遵守。意図的なクリック誘導禁止。 |
| アプリ内ブランド (店舗名・ロゴ) | 公式ロゴ使用時は各ブランドの使用許諾（明示 or 提供媒体の利用規約）を確認。写真を利用する場合は著作権者の許諾が必要。 |

## 7. ドキュメント化・運用フロー
1. 上記骨子をもとに正式な利用規約・プライバシーポリシーを Markdown / HTML で起草し、Gitで管理。
2. 広告文言やクリエイティブは審査表を作成し、景表法チェックリスト（誇大表示、優良誤認、二重価格表示等）を適用。
3. プラットフォームポリシーの更新を半期ごとに確認し、影響があれば速やかにドキュメントをアップデート。
4. ユーザー向けに問い合わせ窓口メールを用意し、データ開示・削除依頼に5営業日以内で対応。
5. 将来的に一般公開・課金を行う場合は、特商法表示・資金決済法・電気通信事業の届出など追加要件を検討。
